# Series Measurement Plot Interpretation

This manual explains how to read and interpret the plots generated by:

```bash
axis workspaces render-series-plots <workspace> --series <series-id>
```

It complements:

- [Series Measurement Artifacts](series-measurement-artifacts-manual.md)
- [Experiment Series](experiment-series-manual.md)

The focus here is not on file layout, but on **meaning**:

- what each plot shows
- what question it helps answer
- how to avoid misreading it

## 1. Plot Families

AXIS currently generates four broad families of measurement plots.

| Category | Source | Scope | Typical question |
|---|---|---|---|
| Framework series overview | framework | whole series | Which experiments look promising overall? |
| Framework per-experiment comparison | framework | one experiment | How do paired episode outcomes differ inside this experiment? |
| System series plots | system extension | whole series | Is the mechanism active, stable, or drifting across the series? |
| System per-experiment plots | system extension | one experiment | What internal mechanism profile explains one concrete experiment? |

Two example workspaces are used throughout this manual:

- `workspaces/system_aw-baseline/series/curiosity-arbitration`
- `workspaces/system_cw_vs_aw/series/system-parameter-variations`

## 2. Reading Strategy

The fastest reliable workflow is:

1. Start with **framework series overview** plots.
2. Then inspect **framework per-experiment comparison** plots for the most interesting experiments.
3. Only after that move into **system-specific** plots.

This order matters because:

- framework plots tell you **whether something happened**
- system plots help explain **why it may have happened**

If you start with system-specific plots too early, it is easy to over-interpret
mechanism activity that does not actually change outcome.

## 3. Framework Series Overview Plots

These plots live under:

- `measurements/plots/series-overview/`

They are generated by the framework and can appear in both:

- `single_system`
- `system_comparison`

When you read the labels, note:

- `single_system` uses `current` vs `baseline`
- `system_comparison` uses `candidate` vs `reference`

### 3.1 `survival-rates.png`

What it shows:

- the fraction of paired episodes that reached the configured horizon
- one bar for each side

How to read it:

- higher bar = more episodes reached `max_steps`
- this is the cleanest top-level success metric

What it does **not** show:

- whether a system usually dies only slightly earlier
- whether both systems mostly fail before the horizon

Interpretation use:

- use it first for a fast “who wins on horizon success?” scan
- do **not** use it alone when many runs are sub-horizon failures

Example:

![Survival Rates](screenshots/series-measurement-plots/framework-series/survival-rates.png)

### 3.2 `paired-survival-counts.png`

What it shows:

- how many paired episodes ended with:
  - left side longer
  - right side longer
  - equal total steps

How to read it:

- this is a **paired** view, not a survival-rate view
- `candidate longer` can increase even if neither side reaches the horizon

Interpretation use:

- compare it against `survival-rates`
- if `survival-rates` are similar but one side is often `longer`, then the
  interesting signal is likely **lifespan robustness**, not horizon success

Example:

![Paired Survival Counts](screenshots/series-measurement-plots/framework-series/paired-survival-counts.png)

### 3.3 `trajectory-vs-survival.png`

What it shows:

- x-axis: mean trajectory distance
- y-axis: survival delta

How to read it:

- farther right = more behavioral divergence
- above zero = left side survives better
- below zero = right side survives better

Interpretation use:

- ask whether behavioral change is:
  - useful
  - neutral
  - harmful

Typical pattern:

- large trajectory distance with near-zero survival delta:
  the system behaves differently, but not usefully

Example:

![Trajectory vs Survival](screenshots/series-measurement-plots/framework-series/trajectory-vs-survival.png)

### 3.4 `efficiency-vs-survival.png`

What it shows:

- x-axis: net energy efficiency
- y-axis: survival delta

How to read it:

- farther right = more efficient energy conversion / retention
- above zero = survival advantage

Interpretation use:

- check whether local metabolic efficiency turns into global robustness
- useful when comparing foraging-heavy vs exploration-heavy settings

Warning:

- better efficiency does not guarantee higher survival if the system still
  collapses structurally elsewhere

Example:

![Efficiency vs Survival](screenshots/series-measurement-plots/framework-series/efficiency-vs-survival.png)

### 3.5 `series-progression.png`

What it shows:

- several key metrics over experiment order on one shared y-axis

Typical curves:

- death rate
- mean final vitality
- net energy efficiency
- survival rates

How to read it:

- use it for directional trends across the series, not for exact values
- look for:
  - smooth progression
  - sharp regime shifts
  - local outliers

Interpretation use:

- especially useful for parameter sweeps where experiments are intentionally
  ordered from weaker to stronger settings

Example:

![Series Progression](screenshots/series-measurement-plots/framework-series/series-progression.png)

### 3.6 `paired-outcome-categories.png`

What it shows:

- the paired outcomes split into six analytically important categories:
  - both reached horizon
  - left side reached only
  - right side reached only
  - both died, left side longer
  - both died, right side longer
  - both died equal

How to read it:

- this plot separates:
  - **horizon success**
  - **sub-horizon advantage**

Interpretation use:

- use this when `survival-rates` feel too binary
- if many counts are in `both died, left side longer`, the system may still be
  getting more robust even without more horizon successes

Example:

![Paired Outcome Categories](screenshots/series-measurement-plots/framework-series/paired-outcome-categories.png)

### 3.7 `horizon-vs-lifespan-delta.png`

What it shows:

- x-axis: survival-rate delta
- y-axis: mean total-steps-lived delta

How to read it:

- top-right:
  better horizon success and longer-lived
- top-left:
  worse horizon success, but longer-lived
- bottom-right:
  more horizon success, but shorter-lived on average elsewhere
- bottom-left:
  broadly worse

Interpretation use:

- this is the cleanest “binary survival vs qualitative robustness” separator
- especially useful for `A+W` vs `C+W`

Example:

![Horizon vs Lifespan Delta](screenshots/series-measurement-plots/framework-series/horizon-vs-lifespan-delta.png)

### 3.8 `sub-horizon-advantage.png`

What it shows:

- only pairs where **both** sides failed before the horizon
- within that restricted set:
  - left side longer
  - right side longer
  - equal
- plus a small `Δ=` annotation for mean steps delta

How to read it:

- this plot intentionally ignores all horizon-reaching wins
- it isolates whether one side is more resilient **inside failure cases**

Interpretation use:

- use this when you suspect:
  - better regulation
  - longer viability
  - partial prediction benefit
  even though survival rate looks flat

Example:

![Sub-Horizon Advantage](screenshots/series-measurement-plots/framework-series/sub-horizon-advantage.png)

## 4. Framework Per-Experiment Comparison Plots

These plots live under:

- `measurements/experiment_N/plots/experiment-comparison/`

They zoom into **one experiment** instead of the whole series.

Example set here comes from:

- `workspaces/system_cw_vs_aw/series/system-parameter-variations/measurements/experiment_2`

### 4.1 `paired-steps-delta-hist.png`

What it shows:

- histogram of `candidate/current total steps - reference/baseline total steps`

How to read it:

- mass near zero = mostly tied lifespan
- mass right of zero = left side tends to last longer
- fat tails = a few very large wins/losses

Interpretation use:

- tells you whether an apparent lifespan advantage is broad or carried by a few
  outlier episodes

Example:

![Paired Steps Delta Histogram](screenshots/series-measurement-plots/framework-experiment/paired-steps-delta-hist.png)

### 4.2 `paired-final-vitality-delta-hist.png`

What it shows:

- histogram of final vitality differences

How to read it:

- right-shift = left side tends to end in stronger metabolic condition
- left-shift = right side tends to end stronger

Interpretation use:

- useful when total steps are similar but end-state quality differs

Example:

![Paired Final Vitality Delta Histogram](screenshots/series-measurement-plots/framework-experiment/paired-final-vitality-delta-hist.png)

### 4.3 `episode-outcomes-strip.png`

What it shows:

- one dot per paired episode
- y position encodes:
  - left side longer
  - equal
  - right side longer

How to read it:

- fast visual summary of whether the experiment is:
  - balanced
  - noisy
  - systematically one-sided

Interpretation use:

- useful before reading histograms in detail

Example:

![Episode Outcomes Strip](screenshots/series-measurement-plots/framework-experiment/episode-outcomes-strip.png)

### 4.4 `mismatch-vs-outcome.png`

What it shows:

- x-axis: action mismatch rate
- y-axis: total steps delta

How to read it:

- farther right = more behavior changed
- above zero = left side lived longer

Interpretation use:

- excellent for asking:
  “Does behavior change actually buy longer life?”

Typical pattern:

- high mismatch but near-zero outcome:
  mechanism changes policy, but not productively

Example:

![Mismatch vs Outcome](screenshots/series-measurement-plots/framework-experiment/mismatch-vs-outcome.png)

### 4.5 `trajectory-distance-distribution.png`

What it shows:

- distribution of mean trajectory distances across episode pairs

How to read it:

- larger values = more spatial/behavioral divergence
- compact distribution = consistent effect
- very spread distribution = unstable or context-specific effect

Interpretation use:

- useful together with `mismatch-vs-outcome`

Example:

![Trajectory Distance Distribution](screenshots/series-measurement-plots/framework-experiment/trajectory-distance-distribution.png)

### 4.6 `steps-lived-distribution.png`

What it shows:

- box plot of actual steps lived for both paired sides

How to read it:

- compare medians, spread, and overlap
- this is often easier to read than a delta histogram when both distributions
  are wide

Interpretation use:

- strongest single per-experiment plot for “one side lasted longer, even though
  neither consistently reached the horizon”

Example:

![Steps Lived Distribution](screenshots/series-measurement-plots/framework-experiment/steps-lived-distribution.png)

## 5.2 `system_c` Series Plots

Plot types currently implemented:

- `c-prediction-error-profile.png`
- `c-trace-balance.png`
- `c-comparison-impact.png`

How to interpret them:

- `c-prediction-error-profile`
  asks whether prediction error stays high, drops, or changes sign across the series
- `c-trace-balance`
  asks whether confidence and frustration traces are stabilizing or fighting each other
- `c-comparison-impact`
  asks whether stronger prediction modulation actually improves outcome

Current note:

- this manual does not yet bundle a dedicated `system_c` screenshot because the
  currently available example workspaces for this documentation are `system_aw`
  and `system_cw`
- conceptually, read `system_c` prediction plots the same way you read the
  prediction-facing parts of the `system_cw` plots below, just without the
  added curiosity/world-model channel

## 7. Common Interpretation Mistakes

### Mistake 1: Treating survival rate as the whole story

Fix:

- also read:
  - `paired-outcome-categories`
  - `sub-horizon-advantage`
  - `steps-lived-distribution`

### Mistake 2: Treating behavioral divergence as benefit

Fix:

- compare:
  - `trajectory-vs-survival`
  - `mismatch-vs-outcome`
  - `prediction-impact-vs-survival`

### Mistake 3: Reading system plots without framework context

Fix:

- start with framework plots
- then use system plots as mechanism explanation

### Mistake 4: Over-interpreting one experiment in isolation

Fix:

- first place the experiment in:
  - `series-progression`
  - `survival-rates`
  - `paired-survival-counts`
