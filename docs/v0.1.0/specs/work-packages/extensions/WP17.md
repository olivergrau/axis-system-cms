# **WP17 Implementation Brief – Extended Cell Regeneration Modes**

## **Context**

We are extending the **AXIS System A Experimentation Framework** on top of the completed baseline runtime and experimentation stack.

At this point, the system already supports:

* deterministic world representation
* local observation construction
* hunger-driven policy behavior
* transition-based cell regeneration
* run / experiment execution
* OFAT-based parameter variation
* persistence, resume, and CLI access

The current regeneration behavior is intentionally simple:

* all non-obstacle cells are treated as regeneration-capable
* each such cell regenerates by a fixed global rate
* resource values are clipped to the valid range

This is acceptable for the minimal baseline, but it produces an overly homogeneous environment. In practice, it means that nearly the entire traversable world is a latent energy field. This weakens spatial structure and reduces the ecological difficulty of the environment.

The world specification already requires spatial heterogeneity and stable environmental structure rather than uniform energy availability across the full grid . Therefore the runtime now needs a more expressive but still minimal regeneration model.

---

## **Objective**

Implement **WP17 – Extended Cell Regeneration Modes**.

The goal of this package is to introduce **multiple deterministic regeneration modes** for the world, while preserving the current baseline behavior as one supported option.

This package must provide:

1. a configurable regeneration mode system
2. preservation of the current global regeneration mode
3. a new sparse regeneration mode in which only a fixed subset of cells is regeneration-capable
4. deterministic world initialization of regeneration-capable cells
5. full integration with existing runtime, experiment config, OFAT usage, and tests

The first new sparse mode shall support the requirement:

> exactly **17% of all grid cells** are regeneration-capable.

These eligible cells must be fixed for a given world initialization and must not change dynamically during the run.

---

## **Scope**

Implement only the following.

---

### **1. Regeneration Mode as Explicit Configuration**

Introduce an explicit world-level regeneration mode.

At minimum, support:

* `all_traversable`
* `sparse_fixed_ratio`

Example configuration shape:

```json
"world": {
  "grid_width": 10,
  "grid_height": 10,
  "resource_regen_rate": 0.05,
  "regeneration_mode": "sparse_fixed_ratio",
  "regen_eligible_ratio": 0.17
}
```

Important:

* the current behavior must remain available through `all_traversable`
* the new mode must be explicit, not inferred indirectly
* invalid mode names must fail explicitly

---

### **2. Preserve Existing Global Mode**

The existing behavior shall be preserved as:

```text
regeneration_mode = all_traversable
```

Semantics:

* every traversable, non-obstacle cell is regeneration-capable
* regeneration is applied uniformly according to the existing deterministic rule
* obstacle cells never regenerate

This preserves backward compatibility with current experiments and avoids breaking baseline behavior.

---

### **3. New Sparse Fixed-Ratio Mode**

Introduce a new mode:

```text
regeneration_mode = sparse_fixed_ratio
```

Semantics:

* only a fixed subset of cells is regeneration-capable
* the subset is determined during world initialization
* the subset remains constant throughout the run
* non-eligible cells never regenerate
* obstacle cells are never eligible

The first required variant is:

```text
regen_eligible_ratio = 0.17
```

Interpretation:

* 17% of all valid grid cells are regeneration-capable
* “valid grid cells” must be defined explicitly

Recommended baseline rule:

* compute eligibility over all **non-obstacle cells**
* select a deterministic subset of size:

$$
N_{eligible} = \mathrm{round}(0.17 \cdot N_{traversable})
$$

Equivalent floor/ceil handling is acceptable, but the rounding rule must be explicit and stable.

---

### **4. Regeneration Eligibility Must Be a Stable World Property**

The system now needs a distinction between:

* a cell’s **current resource value**
* a cell’s **ability to regenerate resource**

This means the implementation must represent regeneration eligibility explicitly.

Acceptable implementation options:

#### **Option A**

Extend cell state with a boolean such as:

```python
is_regeneration_eligible: bool
```

#### **Option B**

Represent eligibility through a fixed cell capability / type field, as long as the semantics remain explicit and inspectable.

Important constraints:

* eligibility must belong to the **world state / world structure**
* eligibility must not be inferred from the current resource value alone
* eligibility must not be recomputed every step
* eligibility must not depend on agent behavior

This is important because the sparse pattern defines part of the stable ecological topology of the world.

---

### **5. Deterministic Eligibility Assignment**

Eligibility assignment must be deterministic under fixed seeds.

Requirements:

* the world generator must use the existing seed context
* identical config + identical seed must produce identical eligible-cell layouts
* different seeds may produce different layouts

A simple acceptable baseline approach:

1. enumerate all traversable non-obstacle cells
2. shuffle deterministically using the configured seed
3. mark the first `N_eligible` cells as regeneration-capable

Important:

* do not use uncontrolled global randomness
* do not distribute eligibility dynamically during execution
* do not let the transition engine choose eligible cells

Eligibility is a **world initialization concern**, not a per-step transition concern.

---

### **6. Regeneration Phase Behavior**

The transition engine must continue to apply regeneration only in **Phase 1**.

However, regeneration must now respect cell eligibility.

#### For `all_traversable`

* regenerate all traversable non-obstacle cells

#### For `sparse_fixed_ratio`

* regenerate only cells marked as regeneration-eligible

The regeneration update rule itself may remain unchanged:

$$
r_{next} = \min(1.0,; r_{current} + \text{regen_rate})
$$

unless the current architecture already uses a different but equivalent bounded rule.

Important:

* this work package is about **where regeneration may occur**, not about changing the mathematical regeneration law
* do not alter action effects, policy behavior, or observation structure beyond what is needed to expose the new world state correctly

---

### **7. Initial Resource State vs Regeneration Capability**

Define the relationship explicitly.

Recommended baseline semantics:

* regeneration capability means the cell **may recover resource over time**
* it does **not** require the cell to start with full resource
* initial resource values may still be zero or configured independently

This distinction matters.
Otherwise sparse regeneration would collapse into a hidden “initial food placement only” mechanism, which is a different concept.

---

### **8. Observation Model Must Remain Unchanged**

The sensor model must continue to expose only:

* traversability
* current resource intensity

It must **not** expose:

* whether a cell is regeneration-eligible
* regeneration mode
* hidden world parameters

This preserves the partial observability rule from the baseline sensor model 

So the agent may experience that some places recover and others do not, but it cannot observe the eligibility map directly.

---

### **9. Configuration Validation**

Add validation rules for the new world parameters.

At minimum:

* `regeneration_mode` must be one of the supported values
* `0 <= resource_regen_rate <= 1`
* if `regeneration_mode == sparse_fixed_ratio`:

  * `0 < regen_eligible_ratio <= 1`

Optional stricter baseline rule:

* reject very small worlds where the ratio produces zero eligible cells unless explicitly allowed

This choice should be explicit.

---

### **10. OFAT Compatibility**

The new regeneration system must integrate cleanly with the experimentation framework.

This means the following paths should be usable in experiment configurations:

* `world.regeneration_mode`
* `world.regen_eligible_ratio`
* `world.resource_regen_rate`

This is important because sparse ecology is exactly the kind of variable that should be sweepable in OFAT experiments.

Example future OFAT:

```json
{
  "experiment_type": "ofat",
  "name": "regen-ratio-sweep",
  "parameter_path": "world.regen_eligible_ratio",
  "parameter_values": [0.05, 0.17, 0.30],
  ...
}
```

---

## **Out of Scope**

Do **not** implement:

* spatially clustered resource biomes
* Perlin noise or Gaussian resource fields
* dynamic creation or deletion of eligible cells during execution
* cell-specific regeneration rates
* history-dependent regeneration
* diffusion between cells
* seasonal or time-varying ecology
* observation of eligibility flags by the agent
* semantic cell labels such as “food patch”

Keep this package minimal and explicit.

---

## **Architectural Constraints**

### **1. World topology must remain explicit**

Eligibility must be stored as part of the world structure, not hidden in transition code.

### **2. Transition phase ordering must remain unchanged**

Regeneration still happens only in Phase 1.

### **3. Sensor purity must remain unchanged**

No leakage of regeneration capability into observation.

### **4. Determinism is mandatory**

Same seed and config must produce same eligible-cell layout and same regeneration behavior.

### **5. Backward compatibility matters**

Existing experiments using the old mode must continue to run unchanged.

## **Testing Requirements**

Add or extend tests for the following.

### **World / Initialization Tests**

* sparse mode marks the correct number of eligible cells
* obstacle cells are never marked eligible
* same seed produces identical eligibility layout
* different seeds can produce different eligibility layouts

### **Regeneration Tests**

* `all_traversable` preserves current behavior
* `sparse_fixed_ratio` regenerates only eligible cells
* ineligible traversable cells do not regenerate
* clipping at `1.0` still holds

### **Observation Tests**

* eligibility does not appear in observation
* two cells with equal current resource but different eligibility are observationally indistinguishable

### **Integration Tests**

* a world in sparse mode runs through full episode execution without breaking policy or transition flow
* OFAT configs can target the new world parameters
* persisted configs/results remain serializable

## **Design Rationale**

The key idea is simple:

The world currently confounds **being traversable** with **being a latent renewable energy source**. That is too strong an assumption.

This WP separates those concerns:

* traversability answers: “can the agent stand there?”
* current resource answers: “how much energy is there now?”
* regeneration eligibility answers: “can this place recover energy over time?”

That separation is mechanistically cleaner and much more useful experimentally.
